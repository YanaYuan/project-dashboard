前三个模块是项目的知识流：信息进来（Sources）→ 被加工成产出（Deliverables）→ 过程中学到的经验沉淀下来（Learnings），反哺下一轮的执行质量。后两个模块是项目的运行基础设施：State 回答"现在怎样"，Logs 回答"过去发生了什么"。

设计原则

每个 Concordia 项目的知识库由 5 个顶级模块 组成。前三个描述项目的内容（输入什么、产出什么、学到什么），后两个描述项目的状态（现在怎样、过去发生了什么）。

所有文件存储在项目的 SharePoint 站点上，团队成员共享。按成员分文件夹的模块（Sources / Deliverables / Learnings）由各成员的 Agent 负责写入，Project Agent 负责去重和索引。

完整目录结构

Project Agent/ │ ├── sources/ ── 输入：项目相关的所有原始信息 │ ├── owner-global/ ── 邮件 / Teams / 日历 / 会议纪要 │ │ ├── {member-alias}/ ── 按成员分文件夹 │ │ └── ... │ ├── sharepoint/ ── SharePoint / OneDrive 文档 │ │ ├── {member-alias}/ ── 按成员分文件夹 │ │ └── ... │ ├── web/ ── 公开网络信息（Project Agent 管理） │ └── _index.md ── 全部 sources 的索引 │ ├── deliverables/ ── 产出：成员交付的文档和工件 │ ├── {member-alias}/ ── 按成员分文件夹 │ └── ... │ ├── learnings/ ── 经验：执行和发现中的可复用教训 │ ├── execution-{member}.md ── 怎么做才对（E 系列） │ └── discovery-{member}.md ── 什么值得做（D 系列） │ ├── state/ ── 当前状态（实时覆盖更新） │ ├── PROJECT.md ── 项目定义 │ ├── todo-list.md ── 候选任务池 │ ├── dashboard.md ── 项目状态快照 │ └── planning/ ── 执行计划 / 时间线 │ └── logs/ ── 历史记录（只追加，不修改） ├── reviews/ ── 每日战略碰撞日志 ├── decisions/ ── 关键决策记录 └── heartbeats/ ── 每小时巡检日志
Sources
输入 — 项目相关的所有原始信息

三个子文件夹，前两个按成员分文件夹（每个成员的 Agent 贡献自己看到的信息），第三个由 Project Agent 统一管理。

子文件夹内容写入者
owner-global/{member}/邮件、Teams 聊天、日历事件、会议纪要各成员 Agent
sharepoint/{member}/SharePoint / OneDrive 上的文档（PPT、文档等）各成员 Agent
web/竞品调研、行业趋势、技术文档Project Agent
去重规则：同一封邮件可能出现在多个成员文件夹中（收/发双方）。Project Agent 保留第一份完整内容，其余替换为 placeholder，标注 duplicate of {member}/filename.md#L42。
Deliverables
产出 — 成员交付的文档和工件

按成员分文件夹。每个交付物附带 _submission.md（含修订历史），经 Owner review 后归档。

示例内容
yinyuan/delegate-meeting-mvp-prd.md代参会 MVP PRD v3.3
jianliu3/autonomous-join-architecture.mdAI 自主入会四层架构方案
aiboliu/pitch-deck.pptxCVP/EVP Pitch Deck
Learnings
经验 — 执行和发现过程中的可复用教训

两种经验，分开存放。每个成员一个文件，采用 Zettelkasten 卡片式管理。

文件内容回答什么写入时机
execution-{member}.mdE001, E002, ...怎么做才对被纠正时 / Daily Reflection
discovery-{member}.mdD001, D002, ...什么值得做Daily Reflection (Part 2)
Zettelkasten 规则：每条经验独立编号，包含：原则（一句话可复用规则）+ 场景（怎么学到的）+ 关联（相关的其他经验编号）。不预设分类，分类从关联中自然涌现。
State
当前状态 — 实时覆盖更新的活文档

回答"项目现在是什么情况"。这些文件会被定期覆盖更新，只保留最新版本。

文件内容更新频率
PROJECT.md项目定义：目标 / 团队 / 交付物 / 时间线偶尔
todo-list.md候选任务池每日（review 追加）
dashboard.md项目状态快照：KPI / 风险 / 进展每小时（heartbeat 刷新）
planning/执行计划 / 关键路径 / 时间线按需
Logs
历史记录 — 只追加、不修改

回答"过去发生了什么"。每个文件创建后不再修改，形成完整的项目运行轨迹。

子文件夹内容频率
reviews/每日战略碰撞日志：发现 / 新增 todo / 风险每日
decisions/关键决策记录：谁 / 什么时候 / 决定了什么按需
heartbeats/每小时巡检日志：任务状态 / 催办 / 升级每小时
